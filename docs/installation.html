Installation Guide

Deploy School ID System to Fly.io or your own infrastructure in minutes.

On This Page

Prerequisites Deploy to Fly.io (Recommended) Self-Hosted Deployment Configuration Google Workspace Integration Local Development

Prerequisites

  • Node.js 20+

    Download from nodejs.org

  • FACTS SIS API Credentials

    Subscription key + API key from your FACTS account

  • S3-Compatible Storage

    For photo storage (Tigris on Fly.io, AWS S3, MinIO, etc.)

Deploy to Fly.io (Recommended)

Fly.io provides the easiest deployment path with built-in Tigris storage for photos.

1 Clone & Install 

git clone https://github.com/Mr-Baker-JPG/school-id.git
ncd school-id
npm install

2 Create Fly App 

fly launch --no-deploy

This creates the app but doesn't deploy yet. Choose a region close to your school.

3 Set Required Secrets 

fly secrets set \
  SESSION_SECRET="$(openssl rand -hex 32)" \
  HONEYPOT_SECRET="$(openssl rand -hex 32)" \
  INTERNAL_COMMAND_TOKEN="$(openssl rand -hex 32)" \
  FACTS_SUBSCRIPTION_KEY="your-subscription-key" \
  FACTS_API_KEY="your-api-key" \
  FACTS_BASE_URL="https://api.factsmgt.com"

4 Create Storage 

fly storage create

This creates a Tigris bucket and automatically sets AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_ENDPOINT_URL_S3, AWS_REGION, and BUCKET_NAME.

5 Deploy 

fly deploy

6 Run Setup Wizard

Visit your app URL followed by /install to configure: 

https://your-app.fly.dev/install

The wizard will guide you through school branding, admin account creation, and optional Google Workspace integration.

Self-Hosted Deployment

Deploy to any Node.js hosting provider (DigitalOcean, Render, Railway, your own server).

1. Clone & Build

git clone https://github.com/Mr-Baker-JPG/school-id.git
cd school-id
npm install
npm run build

2. Configure Environment

cp .env.example .env

Edit .env with your values. See Configuration section below.

3. Initialize Database

npx prisma migrate deploy
npx prisma db seed

4. Start Server

npm start

Then visit http://localhost:3000/install to run the setup wizard.

Configuration

Required Environment Variables

Variable Description
SESSION_SECRET Random string for cookie encryption
HONEYPOT_SECRET Random string for spam protection
INTERNAL_COMMAND_TOKEN Random string for internal API calls
FACTS_SUBSCRIPTION_KEY Your FACTS SIS subscription key
FACTS_API_KEY Your FACTS SIS API key
FACTS_BASE_URL FACTS API URL (usually https://api.factsmgt.com)
AWS_ACCESS_KEY_ID S3 storage access key
AWS_SECRET_ACCESS_KEY S3 storage secret key
AWS_ENDPOINT_URL_S3 S3 endpoint URL
BUCKET_NAME S3 bucket name for photos

Note: School-specific settings (name, colors, branding) are configured through the setup wizard or admin dashboard, not environment variables.

Google Workspace Integration (Optional)

Enable Google OAuth login, Gmail signature management, and automatic photo sync to Google Workspace profiles.

Setup Steps

  1. Create a project in Google Cloud Console
  2. Enable the Google+ API and Admin SDK API
  3. Create OAuth 2.0 credentials (for user login)
  4. Create a Service Account with domain-wide delegation (for signatures + photos)
  5. Enter credentials in the setup wizard or admin settings

Important: Service account domain-wide delegation requires Google Workspace admin access to configure.

Local Development

# Clone the repository
git clone https://github.com/Mr-Baker-JPG/school-id.git
cd school-id

# Install dependencies
npm install

# Copy environment file
cp .env.example .env

# Edit .env with your values

# Initialize database
npx prisma migrate deploy
npx prisma db seed

# Start development server
npm run dev

Visit http://localhost:3000 to see the app, or http://localhost:3000/install to run the setup wizard.

Run Tests

npm run test

Browse Database

npx prisma studio
